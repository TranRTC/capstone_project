=== SLIDE 12 ===
Challenges & Solutions
Engineering Decision Made To Handle specific Technical problems.
01
Real-Time Data Synchronization
Challenge
Keeping multiple browser clients in sync with live sensor data without polling or page refreshes.
Solution
SignalR WebSocket hub with device-group subscriptions. Server pushes to groups on data receipt.
02
High-Volume Time-Series Storage
Challenge
SensorReadings grows to thousands of rows per device per day. Queries must stay fast.
Solution
BIGINT key, composite indexes on (Timestamp, DeviceId), 30-day detail + aggregated archive policy. Consider to work on MongoDB instead of MSSQL
03
Cross-Layer Data Flow Design
Challenge
One sensor reading must flow cleanly: edge device → API → DB → SignalR → React chart.
Solution
Layered architecture (Controller → Service → Repository → DB). SignalR notifies after DB write. DTOs decouple layers.
04
Edge Device Compatibility
Challenge
Supporting Arduino, ESP32, and Raspberry Pi — all with different hardware and communication capabilities.
Solution
Flexible JSON payload format. HTTP POST endpoint for all devices. EdgeDeviceId as universal identifier.
